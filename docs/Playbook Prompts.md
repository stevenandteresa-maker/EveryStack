**EveryStack**

Playbook Generation --- Session A Prompt Guide

*What to upload and what to say to Claude at the start of every sub-phase*

How to Use This Document

This document contains the opening prompts for every sub-phase of EveryStack\'s build. There are 35 sub-phases across 6 phases. For each one, you\'ll find:

**1. FILES TO UPLOAD --- the exact files to attach to the new Claude conversation.**

**2. PASS A-1 PROMPT --- the text to paste as your opening message to generate the playbook.**

**3. PASS A-2 PROMPT --- the text to paste after reviewing the playbook to generate the operator guide.**

**4. EXPECTED DELIVERABLES --- what this sub-phase should produce (update with actuals after each build).**

Starting from sub-phase 1B, every prompt includes a cumulative build summary that tells Claude what has already been built. This gives Claude the context it needs to write accurate playbook preambles and avoid rebuilding existing work.

+-----------------------------------------------------------------------------------+
| **ACTION REQUIRED AFTER EACH SUB-PHASE**                                          |
+-----------------------------------------------------------------------------------+
| IMPORTANT: After each sub-phase is complete, update the \"Expected Deliverables\" |
|                                                                                   |
| section with what was ACTUALLY built. Then copy those actuals into the cumulative |
|                                                                                   |
| summary in all subsequent prompts. The pre-filled summaries are estimates ---     |
|                                                                                   |
| replace them with reality as you go.                                              |
+-----------------------------------------------------------------------------------+

The Pass A-2 Prompt (Same Every Time)

After you review and approve the playbook from Pass A-1, paste this prompt to generate the operator guide:

+-------------------------------------------------------------------------------------+
| **PASTE INTO CLAUDE --- PASS A-2 (SAME FOR ALL SUB-PHASES)**                        |
+-------------------------------------------------------------------------------------+
| The playbook above is approved. Now produce the Pass A-2 operator guide (.docx)     |
|                                                                                     |
| from this playbook, following the Operator Guide Specification in the strategy doc. |
|                                                                                     |
| Include all color-coded containers (blue for prompts, green for git, orange for     |
|                                                                                     |
| checkpoints, gray for explanations), plain-English descriptions for every prompt,   |
|                                                                                     |
| and copy-ready text in every container. The guide should be executable              |
|                                                                                     |
| top-to-bottom with zero technical judgment calls.                                   |
